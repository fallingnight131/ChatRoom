## Interpretation

This result proves that the measurement path works and gives M1/M3 a comparison
point. It does **not** establish production capacity. The clients and server ran
on one Apple Silicon Mac over loopback, one sender waited for its own echo before
sending the next message, and the headless build disables thumbnail generation.
Real WAN latency, TLS, reconnect storms, slow consumers, large rooms, file
traffic, multi-process routing, and database contention are outside this M0
scenario.

The local headless server used Qt 6.9.0 and linked successfully, although that Qt
installation warns that the active macOS 26 SDK is outside its tested build
configuration. Native product CI is pinned to Qt 6.11.1; future comparable
baselines should prefer that supported toolchain and retain the environment
fields in the JSON result.

Treat a later number as comparable only when the scenario fields match. Do not
turn hosted-runner variance into a hard CI latency gate. Capacity targets and a
closed-loop/open-loop load profile belong in the later performance plan.
